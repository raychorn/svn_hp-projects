Events not delivered by Insight Control for vCenter to vCenter Server

Possible cause: Insight Control for vCenter is not configured to deliver events to vCenter Server.

Action: Verify that your systems are configured to deliver events to vCenter Server.

  • Note which events are delivered from Onboard Administrator to vCenter Server as defined in Viewing hardware alert notifications. If any of those events are not delivered to vCenter Server by Insight Control for vCenter, verify that the managed node is an HP ProLiant c-Class server blade and that the corresponding Onboard Administrator is accessible from the default system browser on the system where Insight Control for vCenter is installed.

    For all other events delivered by Insight Control for vCenter to vCenter Server, the HP SNMP agents or HP CIM providers must be installed and configured.

  • Verify that the Onboard Administrator is configured to forward events to Insight Control for vCenter.

    1. Access the Insight Control for vCenter Home Settings page.

    2. Click the Configuration tab.

    3. Verify that OA Settings Forward Events is set to Yes. If No is currently selected, select Yes, and then click Save.
