Cannot configure HP MSA P2000 G3 array when using user-created account

When configuring HP MSA P2000 G3 arrays, an Unable to Authenticate error occurs.

Cause: The WBI does not include an option to enable SMI-S when a new user is created. It has an option to enable FTP, CLI, and WBI only. This is a WBI limitation.

Action: Use the CLI with the manage option to create the user.

For example:create user newUser level manage interfaces cli, ftp, smis, wbi password password
