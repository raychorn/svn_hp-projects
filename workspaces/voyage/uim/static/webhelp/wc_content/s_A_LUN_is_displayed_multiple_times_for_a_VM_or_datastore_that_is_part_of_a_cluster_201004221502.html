LUN displayed multiple times for VM or datastore that is part of a cluster

Possible cause: The Storage Module for vCenter uses an ESXi Host+LUN number as the unique identifier. When a single LUN is presented to multiple hosts that are part of a cluster, the LUN is listed on the Related VMs and Related Datastores pages for each host.

Action: LUNs with the same disk names are duplicates and represent a single disk on the storage array.
